# RwLock

`RwLock` ใน Rust หมายถึง "Read/Write lock" ซึ่งเป็นชนิดของ primitive synchronization มันเป็นเวอร์ชันขั้นสูงของโครงสร้างข้อมูล Mutex ต่างจาก Mutex ซึ่งให้ thread เพียงแค่หนึ่งเท่านั้นสามารถเข้าถึงข้อมูลได้ `RwLock` อนุญาตให้มีหลาย thread อ่านข้อมูลพร้อมกัน แต่มีเพียง thread เดียวที่สามารถเขียนได้ หากมี thread ที่เขียนข้อมูล ไม่มี thread อื่นสามารถอ่านหรือเขียนได้ โครงสร้างสำหรับการประกาศ `RwLock` คือ `let lock = RwLock::new(value)` โปรดจำไว้ว่าข้อเสียของ `RwLock` คือความขัดแย้งของการล็อก จึงต้องดูแลการเข้าถึงอ่าน-เขียนอย่างเหมาะสม การใช้ `RwLock` อย่างเหมาะสมสามารถเพิ่มประสิทธิภาพให้กับโหลดการทำงานที่มีการอ่านหนักมากขึ้น
